Smart Tracker MVP - полный прогресс с нуля
Дата обновления: 2026-04-22

1) Старт проекта и базовый каркас
- Создана структура проекта FastAPI:
  - app/api/routes
  - app/core
  - app/db
  - app/models
  - app/services
  - app/scheduler
  - app/bots
  - app/templates
  - app/static/css
  - scripts
  - alembic
- Добавлены базовые зависимости (FastAPI, SQLAlchemy, Jinja2, Alembic, aiogram, APScheduler и др.).
- Настроен запуск веб-приложения и базовые маршруты.

2) Первая версия доменной модели
- Реализованы основные сущности:
  - users (пользователи и роли)
  - tasks (задания)
  - progress (история статусов заданий)
  - reminders (напоминания)
- Реализованы роли:
  - teacher
  - student
  - parent
- Реализован жизненный цикл задачи:
  - new -> in_progress -> done / overdue

3) Базовый веб-интерфейс
- Добавлены страницы:
  - главная
  - кабинет учителя
  - кабинет ученика
  - кабинет родителя
  - демо-страница
- Добавлены формы создания и управления задачами.
- Реализованы фильтры по статусам и карточки задач.

4) Telegram интеграция
- Добавлен Telegram-бот на aiogram.
- Реализованы команды:
  - /start
  - /tasks
  - /link <code>
- Реализована отправка сообщений пользователю в Telegram через API.
- Реализована привязка Telegram аккаунта к веб-профилю по одноразовому коду.

5) Авторизация и регистрация
- Реализованы:
  - регистрация (логин, email, пароль, подтверждение пароля, роль)
  - вход по логину/паролю
  - сессии через SessionMiddleware
  - выход
- Добавлена проверка уникальности username/email.
- Добавлено хеширование пароля.

6) Подтверждение email и SMTP
- Реализована генерация токена подтверждения email.
- Добавлена страница подтверждения email по ссылке.
- Реализована отправка письма подтверждения через SMTP.
- Добавлены проверки валидности SMTP-конфигурации и сообщения об ошибках.
- Добавлен режим, при котором можно ослабить обязательность подтверждения email через настройку.

7) Telegram 2FA при входе
- Реализован двухэтапный вход:
  - пароль
  - одноразовый код в Telegram
- Добавлена страница ввода кода 2FA.
- Добавлена проверка валидности chat_id для отправки кода.
- Обработаны кейсы истечения кода/ошибки доставки.

8) Alembic и миграции БД
- Подключен Alembic.
- Созданы и применены миграции:
  - 0001_initial_schema
  - 0002_task_ai_metadata
  - 0003_user_preferences_and_links
- Seed-скрипт переведен на сценарий с миграциями.

9) AI-контур и улучшение обработки заданий
- Добавлен сервис ai_gateway:
  - нормализация входа
  - парсинг черновика задания
  - интеграция с локальным Ollama (с fallback)
- В модель задач добавлены AI-поля:
  - source_type
  - parse_confidence
  - parse_raw_input
  - ai_suggestions_json
- Добавлен API endpoint для AI parse draft.
- В форме учителя добавлен AI-preview перед публикацией задания.

10) Политика напоминаний и планировщик
- Реализована policy-логика напоминаний (pre-deadline окна + overdue).
- APScheduler интегрирован с проверкой просроченных задач.
- Добавлена защита от дублей напоминаний через workflow-логику.

11) Демо-цепочка и стабилизация демо
- Реализован demo flow для конкурсного сценария.
- Исправлены проблемы редиректов и 403/401 для веб-роутов.
- Добавлен безопасный маршрут перехода в демо-кабинеты по ролям.
- Добавлено логирование для диагностики проблем регистрации/авторизации/демо.

12) UI/UX улучшения
- Выполнен редизайн интерфейса:
  - новая типографика
  - карточки и KPI-блоки
  - улучшенная визуальная иерархия
  - responsive поведение
- Добавлены визуальные индикаторы статусов задач.
- Добавлены улучшенные формы и экранные состояния.

13) Настройки пользователя (V1)
- Добавлена модель user_preferences:
  - theme (light/dark)
  - accessibility_mode
  - anti_procrastination_mode
  - quiet_hours
  - free_time
  - pomodoro_focus_minutes
  - pomodoro_break_minutes
- Реализовано сохранение настроек из профиля.
- Включено применение темы и accessibility режима на уровне UI.
- Pomodoro значения подключены к таймеру ученика.

14) Привязка ученик-родитель с подтверждением
- Добавлена модель student_parent_links со статусами:
  - pending
  - confirmed
  - rejected
- Реализован сценарий:
  - ученик отправляет запрос родителю
  - родитель подтверждает или отклоняет в профиле
- При создании задачи учителем добавлена автоподстановка подтвержденного родителя.

15) Уведомления по запросу привязки
- После запроса на привязку:
  - родителю отправляется уведомление в Telegram (если привязан)
  - родителю отправляется email уведомление
- После решения родителя:
  - ученику отправляется Telegram уведомление о результате
  - ученику отправляется email уведомление о результате
- Для этого добавлен универсальный helper отправки обычных email (send_plain_email).

16) Профиль и безопасность
- Добавлены операции профиля:
  - смена класса
  - смена email (с повторной верификацией)
  - отвязка Telegram
  - удаление аккаунта
- Ограничено правило: смена класса доступна только ученику.

17) Выдача задания учителем по классу
- В кабинете учителя реализован сценарий:
  - выбор класса
  - выбор режима выдачи:
    - всему классу
    - конкретному ученику
- В режиме "всему классу" создаются задачи каждому ученику класса.
- В режиме "конкретному ученику" выбор ученика фильтруется по выбранному классу.
- После создания показывается success-плашка с количеством созданных задач.

18) Доступность и тёмная тема (доработка)
- Исправлено выравнивание блока настроек.
- Улучшен контраст в dark theme:
  - подписи
  - поля ввода
  - placeholder
  - time picker индикаторы
- Улучшено поведение формы на мобильных экранах.

19) Качество и проверки
- На этапах изменений выполнялись:
  - compileall для проверки синтаксиса
  - проверка линтера измененных файлов
  - применение миграций Alembic
- Исправлялись ошибки интеграций (SMTP, Telegram 2FA, редиректы в демо).

20) Текущее состояние
- Проект представляет рабочий конкурсный MVP:
  - web + Telegram
  - role-based сценарии
  - регистрация/вход/безопасность
  - подтверждаемая связка ученик-родитель
  - выдача задач по классу
  - базовые AI-функции и демо-контур
- Основа готова для следующего шага:
  - более глубокая привязка user preferences к политике уведомлений
  - расширение аналитики/истории и UX polishing.
